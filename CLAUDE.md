# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Purpose

This is a non-code, content-only project. It is a collection of lifestyle and personal development documents focused on simple living, productivity, and habit formation. There are no build steps, tests, or dependencies.

## File Roles

- **`simplify_your_life.md`** — The core reference document. Actionable tips organized into four categories: Physical Space, Mental Clarity, Finances & Shopping, Time & Habits, and Digital Life.
- **`blog_posts_ideas.md`** — Five blog post concepts derived from `simplify_your_life.md`, each expanding a specific tip into a full article premise.
- **`homelab_starting_kit.md`** — A beginner's guide to setting up a Raspberry Pi homelab. Covers hardware, OS, networking basics, recommended services (Pi-hole, Docker, Portainer, Vaultwarden, Home Assistant), and a day-by-day action plan.
- **`gemini_cli_architecture_overview.md`** — A reference document on Gemini CLI agent/skill/tool architecture (informational; not specific to this project's content).
- **`GEMINI.md`** — The equivalent of this file for Gemini CLI. Keep both in sync if project-level guidance changes.
- **`AGENTS.md`** — Contributor guidance for Codex-style tooling. Documents project structure, naming conventions, and agent-specific instructions. Keep aligned with `CLAUDE.md` and `GEMINI.md`.

## Working in This Repo

New content should follow the existing Markdown conventions (ATX headings, bullet lists with `**bold label:**` for tips). When adding blog post ideas, tie them back to a specific tip or section in `simplify_your_life.md`.
